# AE AI Chat

A CEP panel for Adobe After Effects that lets you chat with AI providers directly inside the app. The panel can read project and composition context, capture screenshots, run quick actions, and save temporary ExtendScript actions generated by the model.

[![Screen recording of AI Chat drawing a star](screenshots/Screenrecord_AIChat_Star.gif)](screenshots/Screenrecord_AIChat_Star.mp4)

## Releases

Signed `.zxp` builds are attached to GitHub Releases. Install them with [ZXP Installer](https://aescripts.com/learn/zxp-installer/).

To start a new release from a clean, up-to-date `main` branch:

```bash
just release 0.1.2
# or bump the patch version automatically:
just release-bump
```

`just release-bump` increments the patch version from `package.json`, for example `0.1.2` to `0.1.3`. Both release commands bump `package.json`, commit `Release v0.1.2`, create the `v0.1.2` tag, and push the branch and tag. The pushed tag starts the `Release ZXP` GitHub Actions workflow, which typechecks, signs the ZXP with `ZXP_CERT_PASSWORD`, and uploads it to the GitHub Release.

## Architecture

- CEP panel running inside After Effects with Chromium + Node.js enabled
- Svelte UI for the chat interface
- ExtendScript bridge for After Effects project and comp state
- Provider adapters for:
  - Claude API via the Anthropic SDK
  - Claude CLI
  - Codex CLI

## Platform

- macOS-focused
- Windows is not supported
- Node.js 22.13 or newer is required

## Prerequisites

- After Effects with CEP debug mode enabled
- One working provider:
  - Anthropic API key for `Claude API`
  - `claude` CLI installed and authenticated for `Claude`
  - `codex` CLI installed and authenticated for `Codex`

To enable CEP debug mode, use a tool such as [ZXP Installer](https://aescripts.com/learn/zxp-installer/) and turn on debugging in its settings.

## Setup

From the repo root:

```bash
pnpm install
pnpm build
```

Then restart After Effects and open the panel from **Window > Extensions > AE AI Chat (dev)**. (The signed ZXP release appears as **AE AI Chat** — both can coexist in the menu.)

## Scripts

```bash
pnpm dev       # start the Vite dev server

pnpm zxp       # package a ZXP build
pnpm package   # alias for zxp
pnpm debug     # preview the served panel in a browser
pnpm symlink   # run the local CEP symlink action
```

## Providers

Provider selection happens at runtime from the provider picker shown when the panel opens.

- `Claude API`
  - Available when `ANTHROPIC_API_KEY` is set or a key is saved in the panel UI
  - Supports image attachments
- `Claude`
  - Uses the Claude CLI through a child process
  - Does not support image attachments in the current implementation
- `Codex`
  - Uses the Codex CLI through a child process
  - Does not support image attachments in the current implementation

## Usage

- Type a message to ask about the current After Effects project
- Use the model selector in the header to switch models for the active provider
- The panel automatically includes project and comp context such as:
  - project name and path
  - active comp size, duration, frame rate, and layer count
  - selected layers
  - a capped layer stack summary

Quick actions:

- `Screenshot`
  - Captures the current comp frame
  - Saves it under `screenshots/` next to the current `.aep` project
  - Only appears for providers that support image attachments
- `Report`
  - Runs a deeper comp analysis inside After Effects
  - Caches a richer summary including effects and expressions for later prompts
- `Fix`
  - Sends the most recent panel or action error back to the provider for diagnosis
- `AI Action`
  - Re-runs the current temporary AI-generated ExtendScript action
  - Disabled until an action has been staged

## AI Action Protocol

Providers can return a temporary ExtendScript action by appending an `<ai-action>` block to the response.

Example:

```xml
<ai-action run="true">
// ExtendScript ES3 here
</ai-action>
```

Behavior:

- the panel extracts the first `<ai-action>` block from the response
- the script is saved as a temporary session file
- the action can be run immediately when `run="true"`
- additional `<ai-action>` blocks are ignored after the first

Current session files:

- `.session/ai-action.jsx`
- `.session/ai-action.json`

These files are temporary and overwritten during the panel session.

## How It Works

1. The panel gathers After Effects context through the ExtendScript bridge.
2. The selected provider receives the user prompt plus the AE context.
3. CLI providers run through Node.js child processes; `Claude API` uses the Anthropic SDK directly.
4. Responses stream back into the panel UI with status updates.
5. If the response contains an `<ai-action>` block, the panel saves the script and can execute it through ExtendScript.

## Writing Prompts

Use prompt text that tells the model what to build in After Effects right now. The model's job is to return a runnable `<ai-action run="true">` block with ExtendScript ES3 when the user wants the change executed immediately.

**Use imperative voice.** Write prompts as build instructions.

Good: `Build this 4-layer film treatment in the active composition and execute it immediately.`

Bad: `Produce a faithful recreation spec...` / `Describe how this setup should be built...`

**Explicitly request an AI Action.** The model will not use `<ai-action run="true">` unless the prompt references it — "execute immediately" alone is not enough.

Good: `Respond with an AI Action and run it immediately.`

Bad: `Execute it immediately.` (model may fall back to telling the user to run the script manually)

**Specify exact layer and effect details.** Every layer should include name, type, blend mode, effects in order, exact property values, and exact expression strings. Do not leave room for interpretation.

**Keep scope manageable.** Aim for 4 to 6 layers per prompt. Larger treatments should be split into multiple prompts — smaller prompts execute more reliably and are easier to verify.

**Use display names for effects** — write `Gaussian Blur`, not `ADBE Gaussian Blur 2`. The panel already has the verified effect catalog and match-name lookup table.

**Do not include filesystem instructions.** Do not ask the model to write files, save scripts to another repo, commit changes, or print implementation artifacts. The AI Action protocol handles temporary storage automatically.

See `prompts/sample-prompt.md` for a reference example.
