# AE AI Chat

A CEP panel for Adobe After Effects that lets you chat with AI providers directly inside the app. The panel can read project and composition context, capture screenshots, run quick actions, and save temporary ExtendScript actions generated by the model.

[![Screen recording of AI Chat drawing a star](screenshots/Screenrecord_AIChat_Star.gif)](screenshots/Screenrecord_AIChat_Star.mp4)

## Architecture

- CEP panel running inside After Effects with Chromium + Node.js enabled
- Svelte UI for the chat interface
- ExtendScript bridge for After Effects project and comp state
- Provider adapters for:
  - Claude API via the Anthropic SDK
  - Claude CLI
  - Codex CLI

## Platform

- macOS-focused
- Windows is not supported
- Node.js 18-24 is recommended

## Prerequisites

- After Effects with CEP debug mode enabled
- One working provider:
  - Anthropic API key for `Claude API`
  - `claude` CLI installed and authenticated for `Claude`
  - `codex` CLI installed and authenticated for `Codex`

To enable CEP debug mode, use a tool such as [ZXP Installer](https://aescripts.com/learn/zxp-installer/) and turn on debugging in its settings.

## Setup

From the repo root:

```bash
npm install
npm run build
```

Then restart After Effects and open the panel from **Window > Extensions > AE AI Chat**.

## Scripts

```bash
npm run dev       # start the Vite dev server
npm run build     # build the CEP panel into dist/cep
npm run zxp       # package a ZXP build
npm run package   # alias for zxp
npm run debug     # preview the served panel in a browser
npm run symlink   # run the local CEP symlink action
```

## Providers

Provider selection happens at runtime from the provider picker shown when the panel opens.

- `Claude API`
  - Available when `ANTHROPIC_API_KEY` is set or a key is saved in the panel UI
  - Supports image attachments
- `Claude`
  - Uses the Claude CLI through a child process
  - Does not support image attachments in the current implementation
- `Codex`
  - Uses the Codex CLI through a child process
  - Does not support image attachments in the current implementation

## Usage

- Type a message to ask about the current After Effects project
- Use the model selector in the header to switch models for the active provider
- The panel automatically includes project and comp context such as:
  - project name and path
  - active comp size, duration, frame rate, and layer count
  - selected layers
  - a capped layer stack summary

Quick actions:

- `Screenshot`
  - Captures the current comp frame
  - Saves it under `screenshots/` next to the current `.aep` project
  - Only appears for providers that support image attachments
- `Run Analysis`
  - Runs a deeper comp analysis inside After Effects
  - Caches a richer summary including effects and expressions for later prompts
- `Describe`
  - Sends a prompt asking the active provider to describe the current composition
- `Fix Last Error`
  - Sends the most recent panel or action error back to the provider for diagnosis
- `AI Action`
  - Re-runs the current temporary AI-generated ExtendScript action

## AI Action Protocol

Providers can return a temporary ExtendScript action by appending an `<ai-action>` block to the response.

Example:

```xml
<ai-action run="true">
// ExtendScript ES3 here
</ai-action>
```

Behavior:

- the panel extracts the first `<ai-action>` block from the response
- the script is saved as a temporary session file
- the action can be run immediately when `run="true"`
- additional `<ai-action>` blocks are ignored after the first

Current session files:

- `.session/ai-action.jsx`
- `.session/ai-action.json`

These files are temporary and overwritten during the panel session.

## How It Works

1. The panel gathers After Effects context through the ExtendScript bridge.
2. The selected provider receives the user prompt plus the AE context.
3. CLI providers run through Node.js child processes; `Claude API` uses the Anthropic SDK directly.
4. Responses stream back into the panel UI with status updates.
5. If the response contains an `<ai-action>` block, the panel saves the script and can execute it through ExtendScript.
